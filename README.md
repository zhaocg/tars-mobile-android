# Tars Mobile Android

Native Android client for testing Tars mobile relay chat.

This app mirrors the current iOS mobile client:

- Relay-only connection mode.
- Streaming SSE response handling.
- Message submission through `tars-relay`.
- Markdown rendering in `WebView`.
- Controlled `chart` / `echarts` fenced blocks rendered with bundled ECharts.

## Open in Android Studio

Open this folder in Android Studio and run the `app` configuration on a device
or emulator.

The default Relay URL is:

```text
https://tarsrelay.pqcenter.cn
```

## Relay Settings

In the app Settings dialog:

- Relay URL: your deployed relay URL.
- Session ID: default `mobile-main`.
- Relay Token: the token configured on the relay.
- Agent ID: must match local Tars `TARS_MOBILE_RELAY_AGENT_ID`.
- Client ID: generated by the app and used by relay routing.

Local Tars must be started with matching mobile relay settings:

```powershell
$env:TARS_MOBILE_RELAY_BASE_URL = 'https://tarsrelay.pqcenter.cn'
$env:TARS_MOBILE_RELAY_TOKEN = 'same-token-as-relay'
$env:TARS_MOBILE_RELAY_AGENT_ID = 'default'
npm run start
```

## Endpoints Used

- `GET /integrations/mobile/relay/clients/{clientId}/events`
- `POST /integrations/mobile/relay/agents/{agentId}/sessions/{sessionId}/messages`

Messages are submitted as:

```json
{
  "clientId": "android-device",
  "message": "Hello"
}
```

## Troubleshooting

If the app reports HTTP 404, the Relay URL is wrong or the deployed
`tars-relay` does not include the mobile relay endpoints.

If Android reports an SSL handshake failure, first open
`https://tarsrelay.pqcenter.cn/health` in Chrome on the same device. If Chrome
also fails, fix the relay HTTPS certificate chain before debugging the app.
